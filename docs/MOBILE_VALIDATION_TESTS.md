# Insulet Salesforce Mobile Validation Test Procedures

> Generated by Mobile Architecture Validator  
> Date: 2026-03-05  
> Environment: insulet-devint2  
> Scope: Internal user mobile experience validation

---

## Pre-requisites

- iOS device (iPhone 14 or newer) with Salesforce Mobile App installed
- Android device (Pixel or Samsung) with Salesforce Mobile App installed
- Test user credentials with appropriate profiles (Sales, Service, Admin)
- Test data: at least 1 record of each type (Account, Contact, Lead, Opportunity, Case, Training)
- Network throttling capability (Xcode Network Link Conditioner / Chrome DevTools)

---

## TEST 1: OmniStudio FlexCard Validation (P1 Critical)

### Component: NWAdvancedPatientCard on Patient Account Page

**Steps:**
1. Login to Salesforce Mobile App
2. Navigate to Accounts
3. Open a Patient Account record (Record Type = Patient)
4. Verify the NWAdvancedPatientCard FlexCard renders in the left sidebar region
5. Verify all data fields within the FlexCard are populated and readable
6. Test all interactive elements (buttons, links, expand/collapse)
7. Scroll through the page to verify all 25+ related lists load
8. Measure page load time (target: < 5 seconds on WiFi)

**Visibility Rule Checks:**
- Test with user who HAS `NWAdvancedPatientCard` custom permission: FlexCard should show
- Test with user who does NOT have the permission and is NOT System Administrator: FlexCard should hide
- Test with System Administrator profile: FlexCard should show regardless

**Pass Criteria:**
- [ ] FlexCard renders without layout breaks
- [ ] Data loads within 5 seconds
- [ ] Touch interactions work (tap, scroll)
- [ ] Visibility rules evaluate correctly
- [ ] No horizontal scrolling required
- [ ] All related lists accessible via scroll

**Known Risks:**
- OmniStudio FlexCards may not automatically adapt to mobile viewport
- Custom permission visibility rule must evaluate correctly on mobile context
- 93KB page size may cause performance degradation on slow networks

## TEST 4: Large Page Performance (P2 High)

### Test each page on WiFi and 3G:

| Page | Object | Target Load Time (WiFi) | Target Load Time (3G) |
|------|--------|------------------------|----------------------|
| Patient_Account_Record_Page | Account (Patient) | < 5s | < 10s |
| Provider_Account_Record_Page | Account (Provider) | < 5s | < 10s |
| Trainer_Account_Record_Page | Account (Trainer) | < 5s | < 10s |
| Practice_Account_Record_Page | Account (Practice) | < 5s | < 10s |
| Patient_Delegate_Record_Page | Patient_Delegate__c | < 5s | < 10s |
| Patient_Reported_Outcome | CareObservation | < 5s | < 10s |

**Steps for each page:**
1. Open the record on Salesforce Mobile App
2. Start a timer when tapping the record
3. Stop timer when page is fully rendered (all components visible)
4. Record the time
5. Scroll through the entire page to verify all components loaded
6. Check for blank sections or failed component loads
7. Repeat on 3G throttled connection

**Pass Criteria:**
- [ ] Page loads within target time
- [ ] No blank sections or component load failures
- [ ] Scrolling is smooth (no jank/lag)
- [ ] Memory usage does not cause app crash
- [ ] All related lists render (may lazy-load)
